# 採用ファネル設計

## ■ ファネル構造

① スカウト送信  
② 開封  
③ 応募  
④ 書類通過  
⑤ 1次面接  
⑥ 最終面接  
⑦ 内定  
⑧ 承諾  
⑨ 入社  

---

## ■ 各フェーズ定義

| フェーズ | 説明 |
|--------|------|
| スカウト送信 | 候補者にスカウト送信 |
| 開封 | メール開封 |
| 応募 | 応募発生 |
| 書類通過 | 書類選考通過 |
| 面接 | 面接実施 |
| 内定 | 内定提示 |
| 承諾 | 内定承諾 |
| 入社 | 実際の入社 |

---

## ■ KPI計算式

- 開封率 = 開封 / 送信
- 応募率 = 応募 / 開封
- 書類通過率 = 書類通過 / 応募
- 面接通過率 = 次フェーズ / 面接
- 内定率 = 内定 / 最終面接
- 承諾率 = 承諾 / 内定
- 入社率 = 入社 / 応募

---

## ■ 設計思想

- 各フェーズは「数」と「率」で管理する
- フェーズ間の落ち方を重視する

---

## ■ ボトルネック判定基準

| KPI | 閾値 | 判定 |
|-----|-----|------|
| 開封率 | < 40% | スカウト改善 |
| 応募率 | < 10% | 訴求改善 |
| 書類通過率 | < 30% | ターゲットズレ |
| 面接通過率 | < 40% | 面接課題 |
| 承諾率 | < 50% | 条件・魅力不足 |

---

## ■ 重要な考え方

採用は「母集団 × 質」の最適化

- 母集団 → 応募数
- 質 → 通過率

両方をバランスさせる必要がある
